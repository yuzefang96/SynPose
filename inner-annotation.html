SynPose has two annotation types: COCO style and crowdpose style. Both of their data format are inspired by COCO's data format. The annotations are stored using JSON. The following shows the basic data structure of SynPose:

{
"info"
: info,
"images"
: [image],
"annotations"
: [annotation],
"licenses"
: [license],
"categories"
: [category],
}

info{
"year"
: int,
"version"
: str,
"description"
: str,
"contributor"
: str,
"url"
: str,
"date_created"
: datetime,
}

image{
"id"
: int,
"width"
: int,
"height"
: int,
"file_name"
: str,
}

annotation{
"id"
: int,
"image_id"
: int,
"category_id"
: int,
"bbox"
: [x,y,width,height],
"iscrowd"
: 0 or 1,
"keypoints"
: [x1,y1,v1,...],
"num_keypoints"
: int,
}

license{
"id"
: int,
"name"
: str,
"url"
: str,
}

category{
"id"
: int,
"name"
: str,
"supercategory"
: str,
"keypoints"
: [str],
"skeleton"
: [edge],
}

The differences between COCO style and crowdpose style annotation are as follows: 1) In SynPose14(crowdpose style), it has 14 joints but in SynPose17(COCO style), it has 17. So it will affect the length of annotation->keypoints and category->keypoints. In SynPose14, the length of annotation->keypoints should be 3 * 14 = 42. In SynPose17, the number should be 3 * 17 = 51. 2) Joint connection in COCO and crowdpose is different. See Keypoints for more details. This will affect the array in category->skeleton.

We provide a mini batch of our dataset, and it can be obtained from SynPose14 and SynPose17.
